## 🔐 **SECURITY ARCHITECTURE**

### **Client-Side (Browser):**
- AES-256-GCM for transaction encryption
- RSA-4096 for key exchange
- PBKDF2 password hashing (100K iterations)
- HMAC-SHA256 signatures
- Secure local storage encryption
- Device fingerprinting
- Replay attack prevention

### **Server-Side (Backend):**
- PII encryption (NIDA, phone, address)
- Transaction log encryption
- BOT report encryption
- HMAC verification
- Sensitive data masking
- 256-bit API key generation

### **Network:**
- TLS 1.3 for all API calls
- Certificate pinning
- Perfect forward secrecy

### **Time to Break:**
- **AES-256:** Billions of years
- **RSA-4096:** Infeasible (quantum computers needed)
- **PBKDF2:** ~100 years per password with specialized hardware
